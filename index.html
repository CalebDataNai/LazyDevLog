After three projects in a row where the ORM "magic" turned into debugging sessions, I gave raw SQL another chance. Spoiler: it was fine.

I know, I know — this is a takes-that-never-die. But bear with me. My last three projects used Prisma, SQLAlchemy, and Sequelize respectively. Each one hit a point where I was reading the ORM source code to understand why my join wasn't doing what I expected. At that point, I'm already writing SQL in my head — so why not just… write SQL?

The switch was mostly painless. I use a thin query builder for parameterization and call it a day. My migration files are actual SQL files. My queries are readable to anyone who knows SQL, which is everyone.

There are legitimate reasons to use ORMs — teams unfamiliar with SQL, fast prototyping, auto-generated CRUD. But for anything past the happy path, I'm skeptical.
